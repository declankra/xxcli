What happens technically

Uses Tweepy OAuth 1.0a User Context. All 4 keys come from the X Developer Portal.

Validation: creates a tweepy.Client, calls get_me(). If it fails → "did not validate, try again" loop.

If credentials already exist in config → skips with "Credentials already configured."

FRICTION User must manually navigate to X Developer Portal, create an app, and copy 4 separate keys. No OAuth browser flow, no clipboard help.

Git Repository Picker
onboarding.py:156-177 · THE step you're asking about
Found these git repositories: 1. /Users/macbook/Code/ai-agent-project 2. /Users/macbook/Code/my-saas-app 3. /Users/macbook/Code/portfolio-site 4. /Users/macbook/Code/xxcli 5. /Users/macbook/Projects/side-project 6. /Users/macbook/Developer/experiment Pick a repo number or enter a custom path [1]: 4
What happens technically

Scan logic: Checks 5 hardcoded roots (~/Code, ~/Projects, ~/Developer, ~/repos, ~/src) + cwd. For each root, looks at direct children for .git/ dirs. Sorted alphabetically. No depth > 1.

Selection: Pick a number OR type a custom path. Only ONE repo.

Saved as: default_repo in config.yaml

NO CONTEXT Doesn't explain WHY you're picking a repo. What does it do with it? (Answer: builds git context for the LLM prompt so digest scores tweets by relevance to what you're building)

SINGLE REPO Only picks one default_repo. No support for multiple projects or switching.

NO INTELLIGENCE Alphabetical listing of ALL repos. Doesn't check git log recency — a repo you last touched 2 years ago is listed equally with one you committed to today.

FLAT LIST Can find 50+ repos on a dev machine. No grouping, no filtering, no search.

NO GIT ACTIVITY Doesn't show branch, last commit date, or any signal about which repos are active.

What happens technically

Fetches 15 real tweets from your timeline via get_home_timeline(), shows up to 10.

Each response is logged via log_signal() to ~/.xxcli/feedback.jsonl:

  U(seful) → signal_type="keep", score=8, classification="adopt"

  M(aybe) → signal_type="recover", score=6, classification="copy"

  S(kip) → signal_type="discard", score=2, classification="skip"

  N(oise) → signal_type="discard", score=0, classification="skip"

First "useful" tweet → topic extraction via regex → "Prioritizing [topic] for [repo]"

Architecture Summary

Entry point xx setup → cli.py:302 → onboarding.py:run_setup_wizard()
Also triggered by xx digest if credentials are missing (cli.py:391)
X/Twitter auth OAuth 1.0a — 4 keys manually pasted, validated via tweepy.Client.get_me()
OpenAI auth API key manually pasted, validated via check_openai_key()
Repo discovery Scans 5 hardcoded dirs + cwd for .git/ children (depth=1 only)
Repo selection Single repo only. Stored as default_repo in config
Context building context.py reads git log, diff, branch, README, deps from selected repo
Purpose of repo Feeds into LLM prompt → digest scores tweets by relevance to your current work
Calibration 10 tweets rated → feedback.jsonl → used as few-shot examples for future digests
Config storage ~/.xxcli/config.yaml
Feedback storage ~/.xxcli/feedback.jsonl

Known Issues with Current Flow

REPO PICKER No explanation of WHY you're picking a repo or what it does
SINGLE REPO Only stores one default_repo. Can't track multiple projects.
NO RECENCY Lists repos alphabetically. A 2-year-old dead repo = active project you committed to today.
NO GIT INTEL Doesn't show branch, last commit, activity level. You're choosing blind.
NO SWITCHING xx digest --repo . exists but there's no "switch default repo" command or multi-repo digest.
FLAT DUMP 50+ repos on a dev machine → unmanageable numbered list.
CRED UX 4 separate hidden prompts for X API. High friction, easy to mis-paste.
NO PROGRESS No step indicator (1/6, 2/6...). User doesn't know how far along they are.
CALIBRATION Rating 10 tweets is tedious during setup. Could be deferred or shortened.
